Verification Summary
--------------------
1. Backend quote lifecycle
   - `POST /api/requests/{requestId}/quotes` submits a quote for an open request and returns `quoteId`, `threadId`, and `createdAt`
   - `GET /api/requests/{requestId}/quotes` returns requester-owned quote comparison rows with pagination and sort/filter support
   - `PATCH /api/quotes/{quoteId}` updates a submitted quote and increments `version`
   - `POST /api/quotes/{quoteId}/withdraw` transitions submitted -> withdrawn
   - `POST /api/quotes/{quoteId}/select` transitions submitted -> selected, closes the request, and declines other submitted quotes
   - `POST /api/quotes/{quoteId}/decline` transitions submitted -> declined
   - `GET /api/supplier/quotes` returns the current supplier's submitted quote list

2. Domain / projection / thread linkage

   - `ThreadCommandService` creates or reuses a request/supplier thread on first quote submit
