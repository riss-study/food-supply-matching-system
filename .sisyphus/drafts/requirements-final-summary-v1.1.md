## 1. 한 줄 정리

이 프로젝트의 장기 방향은 풀 패키지 플랫폼이지만, 현재 MVP는 **검증 가능한 공급자 탐색 + 의뢰 + 견적 + 메시지 + 관리자 검수**까지의 신뢰형 매칭 코어에 집중한다.

---

## 2. 지금 확정된 것

### 제품 방향

- 장기 비전: 매칭 + 운영 지원 + 커뮤니티/콘텐츠 확장
- MVP 범위: 신뢰형 매칭 코어
- 런칭 기준: 모든 미래 기능 완성 후가 아니라, Phase 1 완료 조건 충족 후

### 사용자/인증

- 요청자: 사업자 기반 사용자 우선
- 공급자: 가입 가능, 검증 상태에 따라 노출/권한 차등
- 인증 처리: MVP에서는 수동 심사 중심
- 신뢰 표시: 검증 상태 배지/표시 제공

### 핵심 기능

- 가입/로그인
- 공급자 프로필
- 검색/필터
- 의뢰 등록
- 견적 제출/비교
- 메시지 스레드
- 관리자 검수

### 기술 기준

- Frontend: React
- Backend: Kotlin Spring Boot
- RDB: MariaDB 또는 MySQL 호환 계열
- Document/Query Store: MongoDB
- Design Direction: CQRS

---

## 3. MVP에 포함하는 것

### 필수 사용자 흐름

1. 요청자 가입 및 의뢰 등록
2. 공급자 가입 및 프로필 작성
3. 관리자 검수 및 상태 변경
4. 공급자 견적 제출
5. 요청자 비교 및 메시지 상담

### 필수 탐색 기능

- 키워드 검색
- 카드 뷰 / 리스트 뷰
- 카테고리 필터
- 지역 필터
- 인증 상태 필터
- 생산 역량 필터
- MOQ 필터
- OEM/ODM 필터

### 필수 커뮤니케이션 기능

- 텍스트 메시지
- 파일 첨부
- 읽음 상태
- 연락처 공유 동의 흐름

---

## 4. MVP에서 빼는 것

아래 항목은 중요하지만 현재 MVP 기준선에서는 제외한다.

- 리뷰/평점 시스템
- 리뷰 기반 정렬/필터
- 지도 기반 탐색
- WebSocket 기반 실시간 채팅
- OCR 자동 검증
- 외부 인증 API 자동화
- AI 추천/챗봇/가격 예측
- 결제/에스크로/전자계약
- 커뮤니티/교육/컨설팅 기능 확장
