## 1. 개요 (Overview)

### 1.1 프로젝트 목표

네이버 카페 "식품제조(food2008)"의 핵심 매칭 기능을 디지털화하여, **식품 제조 의뢰자(브랜드)**와 **제조업체(공급자)**를 효율적으로 연결하는 B2B 매칭 플랫폼을 구축합니다.

### 1.2 1차 개발 목표 (MVP)

**"신뢰할 수 있는 기본 매칭"**

- 최소한의 필수 기능으로 빠른 런칭
- 핵심 가치: **검증된 제조사 정보** + **효율적인 매칭**
- 2~3개월 개발 후 베타 런칭 목표

### 1.3 타겟 사용자

| 구분 | 대상 | 특징 |
|------|------|------|
| **의뢰자** | 스타트업/중소 브랜드 | 소량 생산, 빠른 런칽 필요 |
| **공급자** | 중소 제조사 | HACCP 인증 보유, OEM/ODM 가능 |

## 2. 1차 개발 범위

### 2.1 기능 분류

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | **회원가입/로그인** | 이메일 기반, 역할 선택 (의뢰자/공급자) | P0 |
| 2 | **사업자 인증** | 사업자등록증 인증 (제조사 필수) | P0 |
| 3 | **제조사 프로필** | 인증, 설비, MOQ, 전문분야 | P0 |
| 4 | **검색/필터링** | 카테고리, 지역, 인증, MOQ 필터 | P0 |
| 5 | **의뢰글 작성** | 템플릿 기반 의뢰서 | P0 |
| 6 | **견적 요청/응답** | 의뢰 → 견적 → 선택 | P0 |
| 7 | **기본 채팅** | 1:1 메시지, 파일 공유 | P0 |
| 8 | **연락처 공개** | 매칭 시 연락처 공개 | P0 |

| 9 | **관심 등록/스크랩** | 제조사 찜 기능 | P1 |
| 10 | **리뷰/평가** | 거래 후 평점/리뷰 | P1 |
| 11 | **알림 시스템** | 이메일/푸시 알림 | P1 |
| 12 | **견적 비교** | 다중 견적 비교 테이블 | P1 |
| 13 | **의뢰자 프로필** | 기본 회사 정보 | P1 |

#### 🔴 선택 (Nice to Have) - 2차 이후

| 15 | **에스크로 결제** | 안전 거래 | P2 |
| 16 | **전자계약** | 온라인 계약 체결 | P2 |
| 17 | **프로젝트 관리** | 생산 일정 추적 | P2 |

### 2.2 식품 제조 특화 필수 기능

#### 인증 관리
- HACCP 인증 여부 표시/필터링 (필수)
- ISO 22000, FSSC 22000 표시 (선택)
- 품목제조보고번호 등록 (선택)

#### 제조 정보
- **가공 유형**: 소스, 제과제빵, 음료, 가공식품, 축산물, 수산, 건강기능식품
- **생산 역량**: 월 최대 생산량, 보유 설비
- **MOQ**: 최소주문수량 필수 입력
- **OEM/ODM**: 가능 여부 표시

#### 견적 정보
- 희망 단가 범위
- 납기 가능 기간
- 원재료 제공 여부
- 포장/라벨링 가능 여부

## 3. 사용자 흐름 (User Flow)

### 3.1 의뢰자 흐름

```
[회원가입] → [의뢰자로 역할 선택] → [기본 정보 입력]
    ↓
[홈: 제조사 검색] → [필터 적용] → [제조사 목록]
    ↓
[제조사 상세] → [견적 요청] 또는 [연락처 공개]
    ↓
[견적서 수신] → [비교/선택] → [채팅/연락]
    ↓
[거래 진행] → [완료 후 리뷰]
```

### 3.2 공급자 흐름

```
[회원가입] → [공급자로 역할 선택] → [사업자 인증]
    ↓
[프로필 작성] → [인증 정보 등록] → [MOQ/견적 설정]
    ↓
[대시보드] ← [의뢰/견적 요청 수신]
    ↓
[견적서 작성/전송] → [협상/채팅]
    ↓
[계약 체결] → [생산 진행] → [납품]
```

## 5. 핵심 기능 상세

### 5.1 회원/인증

#### 가입 프로세스
1. 이메일 입력 및 인증
2. 비밀번호 설정
3. **역할 선택**: 의뢰자(브랜드) / 공급자(제조사)
4. 기본 정보 입력
5. **공급자 추가**: 사업자등록증 인증 (필수)

#### 사업자 인증
- 사업자등록증 사진 업로드
- OCR 또는 수동 확인
- 승인 완료 후 활동 가능

### 5.2 제조사 프로필

#### 필수 입력 항목
```yaml
기본 정보:
  - 회사명
  - 대표자명
  - 설립년도
  - 위치 (시/구)
  - 연락처

인증 정보:
  - HACCP 인증 (필수)
  - ISO 22000 (선택)
  - FSSC 22000 (선택)
  - 유기가공식품 (선택)

제조 역량:
  - 전문 분야 (소스/제과/음료 등)
  - 보유 설비
  - 월 최대 생산량
  - MOQ (최소주문수량)

거래 조건:
  - OEM 가능 여부
  - ODM 가능 여부
  - 원재료 제공 가능 여부
  - 포장/라벨링 가능 여부
  - 희망 단가 범위

포트폴리오:
  - 주요 거래처 (선택)
  - 대표 제품 사진
  - 진행 프로젝트 수
```

### 5.3 검색/필터링

#### 검색 필터
- **카테고리**: 소스, 제과제빵, 음료, 가공식품, 축산물, 수산, 건강기능식품
- **지역**: 시/도, 구/군
- **인증**: HACCP, ISO 22000, FSSC 22000
- **MOQ**: 범위 슬라이더 (1,000개 ~ 100,000개+)
- **정렬**: 추천순, 리뷰순, 최신순

#### 결과 표시
- 기본: 카드 뷰 (회사명, 인증, MOQ, 리뷰점수)
- 전환: 리스트 뷰 (상세 정보)

### 5.4 의뢰/견적 프로세스

#### 의뢰글 템플릿
```yaml
의뢰 기본 정보:
  - 제품명
  - 제품 카테고리
  - 희망 생산량 (월)
  - 희망 단가
  - HACCP 인증 필요 여부

상세 요구사항:
  - 원재료 제공 여부
  - 포장/라벨링 여부
  - 납기 요구사항
  - 시제품 필요 여부

추가 정보:
  - 참고 자료 (사진, 문서)
  - 특이사항
  - 희망 연락 방법
```

#### 견적 요청 → 응답 흐름
1. 의뢰자가 제조사 선택 → 견적 요청
2. 제조사가 견적서 작성 (단가, MOQ, 납기)
3. 의뢰자가 견적 비교
4. 선택한 제조사와 채팅/연락
5. 거래 진행

### 5.5 채팅 시스템

#### 기본 기능
- 1:1 실시간 메시지
- 파일 첨부 (사진, 문서)
- 읽음 확인 표시
- 알림 (이메일 + 앱 내)

#### 채팅 시작 조건
- 견적 요청 시 자동 생성
- 또는 제조사 프로필에서 "문의하기"

## 6. UI/UX 가이드라인

### 6.1 디자인 원칙

1. **단순성**: 복잡한 B2B 정보를 직관적으로 표현
