Date: 2026-03-26
Status: Archived

## 1. 목적

이 문서는 현재 제품 UI를 세 가지 관점에서 동시에 분석한 뒤, 서로의 피드백을 조정해서 실제 리디자인 기준으로 쓰기 위한 합의 문서다.

관점은 아래 세 가지다.

- 디자이너 관점
- PM 관점
- 기획자 관점

---

## 2. 디자이너 관점 피드백

### 잘못된 점

- 공통 스타일 체계가 약하다.
- 헤더, 카드, 버튼, 필터, 테이블이 화면마다 제각각이다.
- 중요한 행동과 덜 중요한 정보의 시각 차이가 약하다.
- 제품 톤이 “신뢰형 B2B 서비스”로 느껴지지 않는다.
- 로그인/회원가입은 제품의 첫인상인데 현재는 거의 개발 확인용 폼처럼 보인다.

### 핵심 요구

- 먼저 공통 셸과 공통 요소를 정리해야 한다.
- main-site는 “신뢰 + 탐색 + 거래” 톤이 느껴져야 한다.
- admin-site는 “빠른 판단 + 명확한 상태” 중심의 내부 운영 도구처럼 보여야 한다.

---

## 3. PM 관점 피드백

### 잘못된 점

- 사용자가 처음 들어왔을 때 무엇을 해야 하는지 바로 이해하기 어렵다.
- 메뉴 구조가 역할별 시나리오보다 개발 편의 기준으로 섞여 있다.
- 핵심 전환 흐름(탐색 -> 의뢰 -> 견적 -> 대화)이 화면 구조상 명확히 이어지지 않는다.
- 로그인/회원가입 페이지가 브랜드 첫인상과 전환 설계 역할을 못하고 있다.

### 핵심 요구

- 헤더는 “전체 서비스 이동”만 담당해야 한다.
- 페이지 내부 행동은 페이지 안에서 해결해야 한다.
- 요청자와 공급자의 주요 행동이 메뉴 구조에서 분리되어 보여야 한다.
- 첫 진입 화면은 서비스가 무엇을 하는지 설명해야 한다.

---

## 4. 기획자 관점 피드백

### 잘못된 점

- 글로벌 메뉴, 역할별 메뉴, 페이지 내부 액션이 혼재되어 있다.
- 상태별 안내 문구는 있지만 화면 단위 섹션 구조가 약하다.
- 일부 화면은 “어떤 정보 묶음이 왜 같이 보이는지”가 설계적으로 약하다.
- 로그인/회원가입, 목록/상세, 액션/설명 구역이 명확히 나뉘지 않는다.

### 핵심 요구

- 정보구조(IA)를 먼저 정리해야 한다.
- 공통 헤더 / 역할별 메뉴 / 페이지 헤더 / 본문 섹션 / 보조 액션 구역을 구분해야 한다.
- 각 화면은 “핵심 목적 1개”를 중심으로 재배치해야 한다.

---

## 5. 서로 충돌하는 의견이 있는가?

크게 충돌하지 않는다.

- 디자이너는 “예쁘게”보다 “체계 있게”를 먼저 원한다.
- PM은 “전환과 흐름이 보이게”를 원한다.
- 기획자는 “정보구조를 먼저 정리”하길 원한다.

즉, 세 관점 모두 같은 결론으로 수렴한다.

---

## 6. 세 관점이 공통으로 동의한 내용

### 합의 1. 지금 문제는 예쁜 색이 아니라 구조 문제다

- 메뉴가 과하고
- 헤더 역할이 과하고
- 화면 내부 섹션이 분명하지 않다.

### 합의 2. 먼저 손봐야 하는 건 공통 셸과 첫 진입 화면이다

- main-site 공통 헤더
- admin-site 공통 헤더
- 로그인
- 회원가입

### 합의 3. 역할별 메뉴를 분리해야 한다

- 비로그인 사용자 메뉴
- 요청자 메뉴
- 공급자 메뉴
- 관리자 메뉴

이게 섞이면 사용자는 현재 자신이 뭘 할 수 있는지 이해하기 어렵다.

### 합의 4. 핵심 거래 흐름은 헤더가 아니라 본문에서 유도해야 한다

- 헤더는 길 찾기
- 본문은 행동 유도

이 역할 분리가 필요하다.

---

## 7. 이번 리디자인의 합의 원칙

1. 글로벌 헤더에는 모든 기능을 다 넣지 않는다.
2. 로그인 여부와 역할에 따라 메뉴를 다르게 보여준다.
3. 로그인/회원가입은 제품 첫인상 화면으로 다시 만든다.
4. 각 페이지는 상단에 페이지 목적과 핵심 행동을 분리해 보여준다.
5. 표/필터/카드/폼은 공통 규칙 안에서 움직이게 한다.

---

## 8. 이번 수정의 우선 적용 범위

### 1차 수정

- main-site 공통 헤더 재구성
- main-site 로그인 페이지
- main-site 회원가입 페이지
- admin-site 공통 헤더 정리

### 2차 수정

- 요청자/공급자 역할별 메뉴 노출 정리
- 공지/내 의뢰/보조 화면도 같은 구조로 맞추기

---

## 9. 최종 결론

세 관점이 합의한 결론은 아래 한 줄로 정리된다.

> 지금 UI 문제의 본질은 “디자인 부족”이 아니라 “정보구조와 화면 역할 분리 부족”이며, 따라서 리디자인은 헤더/메뉴/첫 진입 화면부터 다시 잡아야 한다.

## 10. 실행 반영 메모

- 이 합의안 기준으로 1차 리디자인은 실제 코드에 반영됨.
- 특히 헤더/메뉴 구조, 로그인/회원가입, main-site 핵심 거래 흐름, admin-site 핵심 운영 흐름을 우선 적용함.
