## 0. 이 문서는 어떻게 보면 되나

이 문서는 **지금 당장 코딩 들어가기 전에 꼭 필요한 질문만** 모아둔 파일입니다.

- 여기 있는 질문은 지금 답이 없으면 개발자가 구현하다가 제각각 다르게 만들 수 있는 것들입니다.
- 반대로 인프라 업체, CDN, 고도화 기능, 2차 기능 같은 것은 이 문서에 넣지 않았습니다.
- 그런 나중 결정 항목은 계속 `1st-phase-questionnaire-v2.md`에 모아둡니다.

답변 방법:

- 각 질문 아래 `<!-- 답변: -->` 주석에 적어주세요.
- 길게 안 적어도 됩니다.
- 선택지만 골라도 되고, 짧게 이유를 덧붙여도 됩니다.

## 1. 지금 꼭 필요한 질문만

### Q1. 메시지 스레드는 언제 자동으로 생기나요?

쉽게 말하면:

- 사용자가 서로 대화할 수 있는 채팅방을 **언제 생성할지** 정하는 질문입니다.
- 이걸 안 정하면 어떤 화면에서는 견적만 보내고 채팅이 안 생기고, 어떤 화면에서는 너무 빨리 채팅이 열릴 수 있습니다.

추천 기본값:

- **견적이 1개라도 제출되면**, 요청자-공급자-해당 의뢰 조합으로 스레드 1개 생성

선택지:

- [ ] A. 공급자가 첫 견적을 제출하는 순간 자동 생성
- [ ] B. 요청자가 특정 공급자와 상담 시작 버튼을 눌렀을 때 생성
- [ ] C. A와 B 둘 다 허용
- [ ] D. 기타: __________

<!-- 답변: C. A와 B 둘 다 허용 -->

---

### Q2. 요청자(의뢰자)는 사업자 승인 전에도 의뢰를 등록할 수 있나요?

쉽게 말하면:

- 요청자도 사업자 기반으로 받기로 했는데, **사업자 확인 끝나기 전에 글을 올릴 수 있는지** 정하는 질문입니다.
- 이걸 정해야 회원가입 직후 무엇이 가능하고 무엇이 막히는지가 명확해집니다.

추천 기본값:

- **가입은 가능하지만, 사업자 확인 전에는 의뢰 등록 불가**

선택지:

- [ ] A. 가입만 가능, 승인 전에는 의뢰 등록 불가
- [ ] B. 초안 저장만 가능, 실제 공개 등록은 승인 후만 가능
- [ ] C. 승인 전에도 의뢰 등록 가능
- [ ] D. 기타: __________

<!-- 답변: A. 가입만 가능, 승인 전에는 의뢰 등록 불가 -->

---

### Q3. `견적 수정(PATCH)`은 어디까지 허용하나요?

쉽게 말하면:

- 공급자가 이미 낸 견적을 **나중에 다시 고칠 수 있는지**, 고친다면 어디까지 고칠 수 있는지 정하는 질문입니다.
- 이걸 안 정하면 낸 견적을 마음대로 갈아엎을 수 있는지, 새 견적으로 봐야 하는지가 애매해집니다.

추천 기본값:

- **선택되기 전까지만 수정 가능**, 선택/거절/철회 이후에는 수정 불가

선택지:

- [ ] A. 제출 후에는 수정 불가, 새로 다시 내야 함
- [ ] B. 선택되기 전까지만 수정 가능
- [ ] C. 언제든 수정 가능하지만 수정 이력 저장
- [ ] D. 기타: __________

추가로, 수정 가능 항목이 있으면 적어주세요.

- 예: 단가, MOQ, 납기, 메모만 수정 가능

<!-- 답변: B. 선택되기 전까지만 수정 가능, 근데 수정 이력 저장 하자 -->

---

### Q4. 연락처 공유 요청을 한쪽이 철회(revoke)하면 다시 요청(retry)할 수 있나요?

쉽게 말하면:

- 서로 연락처를 공개하기 전에 동의 과정을 거치기로 했는데,
- 한 번 거절되거나 철회되면 **다시 요청 가능한지** 정하는 질문입니다.

추천 기본값:

- **철회 가능 + 나중에 다시 요청 가능**

선택지:

- [ ] A. 철회 가능, 나중에 다시 요청 가능
- [ ] B. 철회 가능, 하지만 같은 의뢰에서는 재요청 불가
- [ ] C. 철회 자체 불가
- [ ] D. 기타: __________

<!-- 답변: A. 철회 가능, 나중에 다시 요청 가능 -->

---

### Q5. 관리자 검수의 `hold / reject / 재제출` 차이를 어떻게 볼까요?

쉽게 말하면:

- 공급자 인증 심사에서 관리자가 내리는 결과를 **어떻게 구분할지** 정하는 질문입니다.
- 이걸 안 정하면 공급자 화면에 어떤 메시지를 보여줄지, 다시 제출이 가능한지, 검색 노출이 어떻게 되는지가 꼬입니다.

추천 기본값:

- `hold`: 추가 확인 필요, 아직 최종 반려 아님
- `reject`: 현재 제출 건 반려
- `resubmission`: reject 또는 hold 이후 수정해서 다시 제출 가능

선택지:

- [ ] A. hold = 검토 보류, reject = 반려, 이후 재제출 가능
- [ ] B. hold 없이 approve/reject만 사용
- [ ] C. hold = 추가서류 요청, reject = 다시 가입 수준의 반려
- [ ] D. 기타: __________

추가로, 사용자에게 보일 문구 톤도 적어주세요.

- 예: "서류 보완이 필요합니다" / "이번 제출은 승인되지 않았습니다"
