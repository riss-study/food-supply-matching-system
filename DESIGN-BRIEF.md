# Food2008 - 디자인 브리프 (Design Brief)

> 작성일: 2026-03-31
> 목적: 외부 디자이너(사람 또는 AI 디자인 도구)에게 전달하기 위한 통합 디자인 가이드
> 프로젝트: food2008-matching (식품 제조 B2B 매칭 플랫폼)

---

## 1. 서비스 개요

### 한 줄 요약

식품 제조를 맡기고 싶은 **요청자**(제조 의뢰 업체)와 제조 역량을 가진 **공급자**(제조 업체)를 연결하는 **신뢰형 B2B 매칭 플랫폼**.

### 핵심 가치

- 단순 업체 소개가 아니라, **검증된 공급자** 정보를 기반으로 한 신뢰형 매칭
- 의뢰 → 견적 비교 → 메시지 상담 → 연락처 공유까지 **하나의 흐름**으로 연결
- 관리자가 뒤에서 **신뢰를 유지**하는 검수/운영 시스템

### 서비스 톤 & 키워드

- **신뢰** — B2B 거래의 핵심. "이 업체를 믿어도 되는가"가 항상 보여야 함
- **전문성** — 식품 제조업 종사자를 위한 도구. 장식보다 정보의 명확함
- **안전한 연결** — 연락처는 상호 동의 후에만 공개. 단계적 신뢰 쌓기
- **효율** — 비교가 쉽고, 판단이 빠르고, 흐름이 끊기지 않는 UX

## 2. 사용자 역할 (4종)

### 2.1 요청자 (Requester)

- **누구**: 제품 제조를 맡기고 싶은 사업자 (식품 브랜드, 유통사, 스타트업 등)
- **핵심 목표**: 조건에 맞는 제조업체를 찾고, 견적을 비교하고, 안전하게 상담
- **주요 흐름**: 가입 → 사업자 승인 → 공급자 탐색 → 의뢰 등록 → 견적 비교 → 메시지 상담 → 연락처 공유
- **핵심 제약**: 사업자 승인 전에는 의뢰 등록 불가

### 2.2 공급자 (Supplier)

- **누구**: 실제 제품을 제조할 수 있는 업체 (식품 공장, OEM/ODM 업체)
- **핵심 목표**: 프로필로 역량을 보여주고, 적합한 의뢰에 견적을 내고, 요청자와 상담
- **주요 흐름**: 가입 → 프로필 작성 → 검증 서류 제출 → 관리자 승인 → 의뢰 피드 확인 → 견적 제출 → 메시지 상담
- **핵심 제약**: 관리자 승인 전에는 검색 노출 불가, 견적 제출 불가

### 2.3 관리자 (Admin)

- **누구**: 플랫폼 운영자
- **핵심 목표**: 빠른 검수 처리, 운영 현황 파악, 공지 관리
- **주요 흐름**: 검수 큐 확인 → 서류 검토 → 승인/보류/반려 → 공지 작성 → 통계 확인

### 2.4 비회원 / 공개 사용자

- 회원가입 없이 공지사항과 일부 공개 공급자 정보를 볼 수 있음

## 4. 핵심 사용자 흐름 (User Journey)

디자인의 최우선 목표는 이 흐름이 **자연스럽게 이어지는 것**입니다.

### 4.1 요청자 메인 흐름

```
가입 → 사업자 승인 → [공급자 탐색] → [의뢰 등록] → [견적 비교] → [메시지 상담] → 연락처 공유
          ↓                  ↓              ↓              ↓               ↓
     상태 확인 화면      필터/검색으로     공개/지정 모드    표 형태로       상호 동의 후
     승인 전 의뢰 불가    빠른 판단 가능    선택             한눈에 비교     연락처 공개
```

### 4.2 공급자 메인 흐름

```
가입 → 프로필 작성 → 검증 제출 → 관리자 승인 대기 → [의뢰 피드] → [견적 제출] → [메시지 응답]
                                      ↓                 ↓              ↓
                                 상태 안내 명확       조건 확인 후     제출 전 확인
                                 재제출 유도          견적 작성        단계 필요
```

### 4.3 관리자 메인 흐름

```
로그인 → [검수 큐] → [검수 상세] → 승인/보류/반려 → [공지 관리] → [통계 확인]
              ↓              ↓
         우선순위 빠른 판단   서류/이력 한눈에
         대기일수 강조        사유 입력 구조화
```

## 5. 화면별 상세 디자인 요구사항

### 5.1 로그인 (화면 1)

- **절대 안 됨**: 개발 테스트용 폼처럼 보이는 것. 기본값이 미리 채워져 있는 것
- **반드시 필요**: 서비스가 무엇인지 느낄 수 있는 첫인상
- **구성**: 왼쪽(또는 상단)에 서비스 소개 영역 + 오른쪽(또는 하단)에 로그인 폼
- **요소**: 이메일 입력, 비밀번호 입력, 로그인 버튼, 회원가입 링크
- **에러 상태**: 로그인 실패 시 명확한 안내 문구

### 5.2 회원가입 (화면 2)

- **핵심**: 요청자/공급자 역할 선택이 명확해야 함
- **구성**: 서비스 소개 + 역할 선택 카드 + 가입 폼
- **역할 선택 카드**: 각 역할이 무엇을 하는지 간단한 설명 포함
- **요소**: 이메일, 비밀번호, 업체명, 역할 선택 (요청자/공급자)

### 5.3 공급자 탐색 목록 (화면 4) — 가장 중요한 화면

- **이 화면이 서비스의 핵심 가치를 전달하는 메인 화면**
- **필터 영역**: 키워드 검색 + 카테고리 / 지역 / 월 생산량 / MOQ / OEM·ODM 필터
  - 식품 제조 비전문가도 필터를 이해할 수 있는 쉬운 언어 사용
  - 필터와 검색을 시각적으로 분리
- **공급자 카드 (목록 아이템)**:
  - 필수 표시: 회사명, 지역, 카테고리, 월 생산량, MOQ, OEM/ODM 가능 여부, **검증 상태 배지**
  - 선택 표시: 대표 이미지/로고
  - **검증 상태와 핵심 역량이 가장 눈에 띄어야 함**
  - "왜 이 공급자를 봐야 하는가"가 카드만 보고 판단 가능해야 함
- **카드/리스트 뷰 전환** 지원
- **페이지네이션**

### 5.4 공급자 상세 (화면 5)

- **목적**: 특정 공급자의 신뢰도와 제조 가능성을 자세히 판단
- **필수 섹션**:
  - 히어로 영역: 회사명 + 검증 배지 + 지역 + 카테고리
  - 제조 역량 요약: 월 생산량, MOQ, OEM/ODM, 원재료 지원, 포장 지원
  - 설비 정보
  - 인증서 목록 (HACCP, ISO 등)
  - 포트폴리오 이미지
  - 회사 소개
- **핵심 CTA**: "이 공급자에게 의뢰하기" 버튼 (사업자 미승인 시 불가 안내)

### 5.5 의뢰 상세 (화면 8)

- **상태별 UI 변화가 핵심**:
  - `draft` (작성 중): 수정/게시 버튼 활성
  - `open` (진행 중): 수정/마감/취소 가능 + 견적 비교 진입
  - `closed`/`cancelled`: 읽기 전용
- **구성**:
  - 히어로: 제목 + 상태 배지 + 핵심 메타 (카테고리, 모드, 생성일)
  - 본문 카드: 수량, 단가 범위, 원재료, 포장, 납기, 비고
  - 견적 현황 카드: 받은 견적 수 + 견적 비교 버튼
  - 지정 공급자 카드 (지정 모드일 때)

### 5.6 견적 비교 (화면 9)

- **비교가 쉬워야 하므로 표(table) 구조가 핵심**
- **컬럼**: 공급자명, 제출일, 예상 단가, MOQ, 납기, 샘플비, 견적 상태
- **행동 버튼**: 상세 보기, 메시지 이동, 견적 선택, 견적 거절
- **중요**: 선택 시 "다른 견적은 자동 거절" 같은 영향 안내 필요
- **상태별 강조**: 제출됨/선택됨/거절됨/철회됨이 시각적으로 구분

### 5.7 메시지 목록 (화면 10)

- **구성**: 상대방 이름/역할, 관련 의뢰 제목, 마지막 메시지 미리보기, 읽지 않은 수, 마지막 활동 시간
- **읽지 않은 대화는 시각적으로 확실히 강조**

### 5.8 대화 상세 (화면 11)

- **이 화면은 실제 거래 신뢰를 쌓는 핵심 화면**
- **구성**:
  - 헤더: 상대방 정보 + 관련 의뢰명
  - 연락처 공유 상태 배너 (4가지 상태)
  - 메시지 영역: 말풍선 + 첨부파일 + 읽음 표시
  - 입력 영역: 텍스트 + 파일 첨부 버튼
- **연락처 공유 상태별 UI**:
  - `not_requested`: 연락처 공유 요청 버튼
  - `requested`: 대기 중 안내 + 철회 버튼
  - `one_side_approved`: 한쪽 승인 안내 + 상대방 승인 대기
  - `mutually_approved`: 양측 연락처 카드 노출 (전화, 이메일)
- **업무 툴이 아니라, 신뢰를 느낄 수 있는 상담 공간**처럼 디자인

### 5.9 검수 큐 (화면 13)

- **관리자가 빠르게 우선순위를 판단할 수 있어야 함**
- **구성**: 상태 필터 탭 + 기간 필터 + 정렬 + 테이블
- **테이블 컬럼**: 회사명, 현재 상태, 제출일, 대기일수
- **"지금 급한 건 무엇인지"가 바로 보여야 함** (대기일수 강조, 상태 색상 구분)

### 5.10 검수 상세 (화면 14)

- **구성**:
  - 회사 기본 정보 요약
  - 제출 파일 목록 (다운로드 가능)
  - 검수 이력 타임라인
  - 결정 영역: 승인/보류/반려 버튼 + 내부 메모 + 사용자 공개 메모
- **`hold`(보류) = 보완 요청, `reject`(반려) = 현재 건 반려** 의미가 명확해야 함
- 재제출 가능하다는 안내 포함

### 5.11 통계 대시보드 (화면 16)

- **구성**:
  - 상단 KPI 카드: 전체 사용자 수, 요청자/공급자/관리자 수
  - 공급자 상태별 분포 (막대/도넛 차트)
  - 검수 대기 현황
  - 의뢰 현황
  - 기간 필터
- **숫자만 나열이 아니라, 카드 + 간단한 시각화로 한눈에**

## 8. 디자인 원칙 & 톤

### 8.1 main-site 톤

- **신뢰형 B2B 서비스** — 깔끔하고 전문적이지만, 차갑지 않은 Warm Neutral 톤
- 참고할 방향: 토스(toss.im)의 정보 정돈감 + 배민(b2b 영역)의 따뜻함 + Notion 같은 구조적 깔끔함
- 과한 SaaS 템플릿 감성은 피할 것
- 식품 제조업이라는 산업 특성상, 너무 테크 느낌보다는 **실무 도구** 느낌

### 8.2 admin-site 톤

- **빠른 판단 + 명확한 상태** 중심의 내부 운영 도구
- 화려함보다 정보 밀도와 상태 가시성 우선
- 참고할 방향: Linear, Retool 같은 운영 도구의 깔끔한 정보 밀도

### 8.3 공통 원칙

1. **정보 위계가 명확할 것** — 제목, 상태, 본문, 보조 정보, 액션 버튼의 시각적 강도 차이
2. **핵심 행동이 설득력 있을 것** — "왜 지금 이걸 눌러야 하는지"가 보여야 함
3. **상태를 색에만 의존하지 않을 것** — 배지 + 색상 + 아이콘 + 텍스트 조합
4. **에러 문구는 "왜 안 되는지" + "어떻게 하면 되는지"를 함께**
5. **어려운 용어 대신 업무 언어** — 식품 제조 비전문가도 이해할 수 있게
6. **각 페이지는 핵심 목적 1개를 중심으로** — 한 화면에서 한 번에 이해

## 13. 현재 디자인 문제점 요약 (개선 필요 사항)

외부 디자이너가 참고할 현재 상태의 핵심 문제:

### 구조적 문제

1. **공통 스타일 시스템이 약함** — 각 화면이 독립적으로 스타일링되어 일관성 부족
2. **정보 위계 부족** — 제목, 상태, 본문, 버튼이 모두 비슷한 강도로 보임
3. **핵심 행동 설득력 부족** — 중요한 버튼이 왜 눌러야 하는지 시각적으로 전달 못함
4. **제품 톤 부재** — "개발 확인 화면"에서 "서비스 화면"으로의 전환 필요

### 화면별 문제

- **공급자 탐색**: 필터 경험이 약하고, 공급자 카드에서 핵심 포인트가 안 보임
- **견적 비교**: 데이터 테이블 느낌만 있고, "비교"되는 느낌이 약함
- **메시지 상세**: 기능은 충분하지만 거래 신뢰를 쌓는 공간으로서의 감정적 톤이 약함
- **로그인/회원가입**: 제품 첫인상 역할을 못함
- **관리자 검수 큐**: 우선순위 판단이 어렵고, 급한 건이 안 보임
- **통계 대시보드**: 숫자 나열 수준이고, 판단 도구로서의 인상이 약함

## 부록: 용어 사전

| 용어 | 설명 |
|------|------|
| OEM | 주문자 상표 부착 생산. 고객 브랜드로 제조 |
| ODM | 제조자 설계 생산. 제조사가 설계까지 담당 |
| MOQ | Minimum Order Quantity. 최소 주문 수량 |
| HACCP | 식품안전관리인증기준. 식품 제조 필수 인증 |
| ISO | 국제표준화기구 인증 |
| FSSC | 식품안전시스템인증 |
| 검수 | 관리자가 공급자 제출 서류를 검토하는 과정 |
| 의뢰 모드: 공개 | 모든 적합 공급자에게 공개되는 의뢰 |
| 의뢰 모드: 지정 | 특정 공급자에게만 보이는 의뢰 |
