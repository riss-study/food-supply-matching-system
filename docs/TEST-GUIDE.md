## 2. 서비스 기동

### 2-1. 인프라 (Docker)

```bash
export DOCKER_HOST=unix:///Users/riss/.colima/default/docker.sock

cd backend
docker compose -f compose.local.mariadb.yml up -d
docker compose -f compose.local.mongodb.yml up -d
```

기동 후 확인:

```bash
docker ps   # mariadb, mongodb 컨테이너가 healthy 상태인지 확인
```

### 2-2. 백엔드

두 서버 모두 `SPRING_PROFILES_ACTIVE=local` 프로파일이 필요합니다.

```bash
cd backend
SPRING_PROFILES_ACTIVE=local ./gradlew :api-server:bootRun &
SPRING_PROFILES_ACTIVE=local ./gradlew :admin-server:bootRun &
```

> **팁:** 첫 기동 시 Gradle 빌드에 1~2분 정도 소요될 수 있습니다. `Started ApiServerApplication` / `Started AdminServerApplication` 로그가 출력되면 기동 완료입니다.

### 2-3. 프론트엔드

```bash
cd frontend/apps/main-site && npx vite --port 5173 &
cd frontend/apps/admin-site && npx vite --port 5174 &
```

### 2-4. 전체 기동 확인 요약

| 서비스 | 확인 방법 | 정상 응답 |
|--------|----------|----------|

| Admin-site | 브라우저에서 http://localhost:5174 접속 | 로그인 화면 렌더링 |

| 3 | 기간 드롭다운 변경 | 선택 기간에 해당하는 데이터로 갱신 |
| 4 | 테이블 행 클릭 | 검수 상세 페이지로 이동 |

#### 4-3. 검수 상세 (승인/보류/거절)

## 9. 알려진 제한 사항 및 주의점

### 환경 관련

- **Colima 소켓 경로**: Docker 명령 실행 전 반드시 `DOCKER_HOST` 환경변수를 설정해야 합니다. 설정하지 않으면 Docker 데몬에 연결되지 않습니다.
- **포트 충돌**: 5173, 5174, 8080, 8081, 13306, 27018 포트가 이미 사용 중이면 기동에 실패합니다. `lsof -i :<포트번호>`로 확인 후 충돌 프로세스를 종료하세요.
- **Gradle 첫 빌드**: 의존성 다운로드로 인해 최초 기동 시 수 분이 소요될 수 있습니다.

### 기능 관련

- **테스트 데이터**: 초기 기동 시 DB 스키마만 생성되며, 시드 데이터가 없을 수 있습니다. 테스트 시나리오 실행 전 테스트 계정과 기초 데이터가 존재하는지 확인하세요.
- **파일 업로드**: 사업자등록증 등 파일 업로드 기능이 로컬 환경에서 제한될 수 있습니다. 업로드 경로 및 저장소 설정을 확인하세요.
- **이메일 발송**: 로컬 환경에서는 실제 이메일이 발송되지 않습니다. 로그에서 발송 내역을 확인하세요.
- **CQRS 구조**: 커맨드(MariaDB)와 쿼리(MongoDB) 모델이 분리되어 있어, 데이터 작성 후 조회에 약간의 지연(eventual consistency)이 발생할 수 있습니다.

### 테스트 수행 시 주의

- 시나리오 테스트는 **순서대로** 수행하는 것을 권장합니다 (회원가입 → 요청자 플로우 → 공급자 플로우 → 관리자 플로우).
- 브라우저 캐시/쿠키로 인한 인증 이슈 발생 시 시크릿 창을 사용하세요.
- 요청자/공급자 동시 테스트 시 **별도 브라우저** 또는 **시크릿 창**을 활용하세요.
- Admin-site에 Main-site 계정(buyer/supplier)으로 로그인을 시도하면 실패해야 정상입니다. 역도 마찬가지입니다.
